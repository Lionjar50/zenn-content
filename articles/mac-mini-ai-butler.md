## Mac miniが「ただの小さいPC」で終わるのはもったいない

Mac miniを買ったものの、持て余している人は多いのではないでしょうか。

自分もそうでした。Apple Silicon搭載のMac mini、省電力で静か。でもやっていることはブラウザとターミナルだけ。これ、24時間稼働させたら面白いことができるのでは？

そう思って **OpenClaw** というAIエージェントフレームワークをセットアップしたら、Mac miniが「AI秘書」になりました。この記事はそのセットアップと、実際に何ができるようになったかの記録です。

---

## OpenClawとは

[OpenClaw](https://github.com/openclaw/openclaw) は、LLM（大規模言語モデル）をバックエンドにしたAIエージェントを、自分のマシン上で24時間動かせるオープンソースのフレームワークです。

特徴をざっくり言うと：

- **常駐型**: デーモンとしてバックグラウンドで動き続ける
- **チャット連携**: Telegram、Discord、WhatsAppなどと接続できる
- **ツール実行**: シェルコマンド、ファイル操作、Web検索、ブラウザ操作が可能
- **スキル拡張**: プラグインのように機能を追加できる
- **スケジュール実行**: cronジョブで定期タスクを回せる

ChatGPTやClaudeの「ウェブ版」との最大の違いは、**自分のマシン上で動く**こと。ファイルにアクセスでき、コマンドを実行でき、外部サービスと連携できます。しかもセッションが途切れても記憶を保持します。

## 使ったもの

| 項目 | 内容 |
|------|------|
| マシン | Mac mini（Apple Silicon / macOS 26） |
| ランタイム | Node.js v25 |
| AIモデル | Claude（Anthropic API） |
| チャット | Telegram Bot |
| 費用 | Anthropic APIの従量課金のみ（月数千円〜） |

Mac miniの消費電力は待機時で数ワット程度。電気代はほぼ気になりません。

## セットアップ手順

### 1. Node.jsのインストール

OpenClawはNode.js上で動きます。

```bash
# nvmでインストール（推奨）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
nvm install --lts
```

### 2. OpenClawのインストール

```bash
npm install -g openclaw
```

### 3. 初期セットアップ

```bash
openclaw onboard
```

対話形式でセットアップが始まります。聞かれるのは主に以下の3つ：

- **AIプロバイダー**: Anthropic（Claude）を選択し、APIキーを入力
- **チャンネル**: Telegram Botを選択し、BotFatherで取得したトークンを入力
- **ワークスペース**: ファイルを置くディレクトリの指定

### 4. 起動

```bash
openclaw gateway start
```

これで常駐デーモンが立ち上がります。Telegramでボットにメッセージを送ると、AIが応答を返すようになります。

## 「セットアップしただけ」で終わらない理由

ここまでなら「ChatGPTのAPI叩いてるだけでは？」と思うかもしれません。

OpenClawが面白いのは、AIに**人格とワークスペースを持たせる**設計です。

### ファイルが記憶になる

ワークスペースには以下のようなファイルがあります：

```
workspace/
├── SOUL.md        # AIの人格・行動原則
├── USER.md        # あなたの情報（名前、タイムゾーンなど）
├── IDENTITY.md    # AIの名前・キャラクター
├── MEMORY.md      # 長期記憶
├── HEARTBEAT.md   # 定期チェック項目
└── memory/
    └── 2026-02-21.md  # 日次の記録
```

AIはセッション開始時にこれらのファイルを読み、自分が誰で、誰のために動いているかを把握します。会話で得た重要な情報はファイルに書き出されるので、翌日のセッションでも文脈が引き継がれます。

LLMは本来ステートレスです。でもファイルシステムを「外部記憶」として使うことで、**疑似的な長期記憶**を実現しています。

### 「秘書」は比喩ではない

自分の場合、`SOUL.md`にこう書いています：

> 最善の行動を自ら判断し提案する。指示を待つのではなく、先を読んで動く。

こう書くと、AIは本当に「秘書」として振る舞い始めます。頼んでいないのにメールを確認し、カレンダーの予定をリマインドし、天気を教えてくれる。人格の設定は、プロンプトエンジニアリングそのものです。

## 実際にやらせていること

### 定期タスクの自動実行

`HEARTBEAT.md`に書いた項目を、30分おきにAIがチェックします：

- システム監視（CPU/メモリ/ディスク）
- ネットワークの疎通確認
- メールの確認
- カレンダーの予定確認

異常があればTelegramに通知が来ます。何もなければ黙っています。この**「異常時だけ話しかける」**動作は、cronジョブ＋通知を自前で組むより遥かに手軽です。

### Webの情報収集

「このURLの内容を要約して」「Zennのトレンド見て」といったリクエストに応えてくれます。ブラウザを操作する機能もあるので、ログインが必要なサービスの操作も可能です。

### ファイル操作と開発補助

ワークスペース内のファイルは自由に読み書きできるので、ちょっとしたスクリプトを書かせたり、設定ファイルを編集させたりもできます。

## ハマったポイント

### APIキーの管理

Anthropic APIキーは `openclaw onboard` で設定しますが、環境変数ではなくOpenClawの設定ファイルに保存されます。`.env`に書いても読まれないので注意。

### Mac miniのスリープ

24時間稼働させるなら、スリープを無効にする必要があります。

```bash
# システム設定 > エネルギー > スリープしない に設定
# またはコマンドで
sudo pmset -a sleep 0
sudo pmset -a disablesleep 1
```

### Telegramボットの作成

BotFatherで `/newbot` してトークンを取得するだけですが、**ボットはDMを先に送ってきたユーザーにしか返信できない**点に注意。最初に自分からボットにメッセージを送る必要があります。

## 気になるコスト

Anthropic APIの従量課金です。Claude Sonnetで日常的な会話＋定期タスクを回して、月のAPI費用は数千円程度。定期タスクの頻度を上げすぎるとトークン消費が増えるので、`HEARTBEAT.md`の項目は必要最小限にするのがおすすめです。

### Mac miniの消費電力と電気代

Mac mini（M4）の消費電力は公式スペック上、**アイドル時 5W、最大負荷時 65W** です。

AIエージェントの待機中はほぼアイドル状態なので、実際の平均消費電力は **5〜10W** 程度。これを24時間365日で計算すると：

| 状態 | 消費電力 | 月間電力量 | 月間電気代（31円/kWh） |
|------|----------|-----------|----------------------|
| アイドル（待機中心） | 5W | 3.6kWh | 約112円 |
| 軽負荷（API応答時） | 10W | 7.2kWh | 約223円 |
| 平均的な利用 | 7W | 5.0kWh | 約155円 |

**月の電気代は100〜200円程度**。エアコンの待機電力より安いです。

参考までに、同じことをクラウドVM（AWS EC2 t4g.mediumなど）でやると月$30〜50かかります。Mac miniは初期投資（84,800円〜）こそありますが、ランニングコストでは圧倒的に有利です。

### トータルコスト

| 項目 | 月額 |
|------|------|
| Mac mini 電気代 | 約150円 |
| Anthropic API | 約3,000〜5,000円 |
| **合計** | **約3,000〜5,000円** |

実質的なコストはほぼAPI利用料です。Mac miniを24時間稼働させること自体のコストは無視できるレベルです。

## まとめ

Mac mini + OpenClawの組み合わせで得られるのは、**「自分専用の、常に動いているAI」**という体験です。

ウェブ版のAIチャットは便利ですが、ブラウザを開いて、質問を打って、回答を待つという流れは「道具を使っている」感覚です。一方でTelegramに常駐するAI秘書は、どちらかというと「同僚にチャットで頼む」感覚に近い。この体験の違いは、使ってみないと分かりません。

技術的には難しいことはしていません。Node.jsが動くマシンがあれば、30分でセットアップできます。Mac miniが眠っている方は、ぜひ試してみてください。
