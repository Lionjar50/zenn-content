**結論から言うと、X APIのFree planではツイート投稿に402エラーが返ります。** Basic planは月額$200。個人開発者には厳しい現実です。

ただ、GAS × OAuth 1.0aで自動投稿システムを作る過程自体は学びが多かったので、コード付きで共有します。

## やりたかったこと

Googleスプレッドシートに「投稿日時」「本文」「画像」を入力しておけば、GASが1分間隔でチェックして自動投稿してくれる仕組みです。

```
スプレッドシート（投稿予約）
  ↓ GASトリガー（1分間隔）
  ↓ 投稿時刻が来た行を検出
  ↓ OAuth 1.0a で署名生成
  ↓ X API v2 で投稿
  ↓ ステータスを「投稿済」に更新
```

スプレッドシートをCMSとして使う発想です。非エンジニアでも投稿予約ができます。

## OAuth 1.0aの署名生成

X APIはOAuth 1.0aを使います。OAuthライブラリに頼らず、GASだけで署名を生成しました。

ここが一番ハマったポイントです。署名の生成手順は以下の通り。

### 1. OAuth パラメータを用意する

```javascript
var oauthParams = {
  oauth_consumer_key: API_KEY,
  oauth_nonce: Utilities.getUuid().replace(/-/g, ''),
  oauth_signature_method: 'HMAC-SHA1',
  oauth_timestamp: Math.floor(Date.now() / 1000).toString(),
  oauth_token: ACCESS_TOKEN,
  oauth_version: '1.0'
};
```

`oauth_nonce` はリクエストごとにユニークな文字列。GASの `Utilities.getUuid()` が便利です。

### 2. 署名ベース文字列を作る

```javascript
// 全パラメータ（OAuth + リクエスト）をソート
var sortedKeys = Object.keys(allParams).sort();
var paramStr = sortedKeys.map(function(k) {
  return encodeRfc3986_(k) + '=' + encodeRfc3986_(allParams[k]);
}).join('&');

// HTTPメソッド + URL + パラメータ文字列を結合
var baseStr = method.toUpperCase() + '&' 
  + encodeRfc3986_(url) + '&' 
  + encodeRfc3986_(paramStr);
```

ポイントは **RFC 3986準拠のエンコード** です。`encodeURIComponent` だけでは不十分で、`!`、`*`、`'`、`(`、`)` も追加でエンコードする必要があります。

```javascript
function encodeRfc3986_(str) {
  return encodeURIComponent(String(str))
    .replace(/!/g, '%21')
    .replace(/\*/g, '%2A')
    .replace(/'/g, '%27')
    .replace(/\(/g, '%28')
    .replace(/\)/g, '%29');
}
```

### 3. HMAC-SHA1で署名する

```javascript
var signingKey = encodeRfc3986_(API_KEY_SECRET) + '&' + encodeRfc3986_(ACCESS_TOKEN_SECRET);

var signatureBytes = Utilities.computeHmacSignature(
  Utilities.MacAlgorithm.HMAC_SHA_1,
  baseStr,
  signingKey
);
var signature = Utilities.base64Encode(signatureBytes);
```

GASの `Utilities.computeHmacSignature` がHMAC-SHA1に対応しているので、外部ライブラリ不要です。

### 4. Authorizationヘッダーを組み立てる

```javascript
oauthParams['oauth_signature'] = signature;

var headerParts = Object.keys(oauthParams).sort().map(function(k) {
  return encodeRfc3986_(k) + '="' + encodeRfc3986_(oauthParams[k]) + '"';
});

return 'OAuth ' + headerParts.join(', ');
```

## ツイート投稿（v2 API）

署名ができれば、投稿自体はシンプルです。

```javascript
function postTweet_(text, mediaIds) {
  var url = 'https://api.twitter.com/2/tweets';
  var payload = { text: text };

  if (mediaIds && mediaIds.length > 0) {
    payload.media = { media_ids: mediaIds };
  }

  var authHeader = buildOAuthHeader_('POST', url, {});

  var response = UrlFetchApp.fetch(url, {
    method: 'post',
    headers: {
      'Authorization': authHeader,
      'Content-Type': 'application/json'
    },
    payload: JSON.stringify(payload),
    muteHttpExceptions: true
  });

  var code = response.getResponseCode();
  if (code !== 201) {
    throw new Error('投稿失敗 (' + code + ')');
  }

  return JSON.parse(response.getContentText()).data.id;
}
```

**注意点：** `buildOAuthHeader_` の第3引数にリクエストボディのパラメータを渡す必要は**ありません**。JSON bodyの場合、OAuth署名にbodyのパラメータは含めません。ここでハマる人が多いようです。

## 画像アップロード（v1.1 API）

画像付きツイートは2段階です。まず画像をアップロードしてmedia_idを取得し、ツイート投稿時に紐づけます。

```javascript
function uploadMedia_(blob) {
  var url = 'https://upload.twitter.com/1.1/media/upload.json';
  var base64Data = Utilities.base64Encode(blob.getBytes());

  var params = { media_data: base64Data };
  var authHeader = buildOAuthHeader_('POST', url, params);

  var response = UrlFetchApp.fetch(url, {
    method: 'post',
    headers: { 'Authorization': authHeader },
    payload: { media_data: base64Data },
    muteHttpExceptions: true
  });

  return JSON.parse(response.getContentText()).media_id_string;
}
```

画像アップロードは **v1.1 API**（`upload.twitter.com`）を使います。v2にはまだメディアアップロードのエンドポイントがありません（2026年2月時点）。

Google Driveに画像を置いておけば、スプレッドシートにファイルIDかURLを書くだけで自動取得されます。

## スプレッドシートの構成

| A列（投稿日時） | B列（本文） | C〜F列（画像1〜4） | G列（ステータス） | H列（エラー理由） | I列（投稿ID） |
|---|---|---|---|---|---|
| 2026/02/18 07:00 | おはようございます☀️ | （DriveのファイルID） | | | |

GASのトリガーが1分ごとに全行をチェックし、「投稿日時 ≤ 現在時刻」かつ「ステータスが空」の行を投稿します。

```javascript
function checkAndPost() {
  var sheet = SpreadsheetApp.openById(SPREADSHEET_ID).getSheets()[0];
  var data = sheet.getDataRange().getValues();
  var now = new Date();

  for (var i = 1; i < data.length; i++) {
    var row = data[i];
    if (String(row[COL_STATUS]).trim() !== '') continue;

    var postDate = new Date(row[COL_DATETIME]);
    if (now >= postDate) {
      // この行を投稿
      var tweetId = postTweet_(String(row[COL_TEXT]).trim(), []);
      sheet.getRange(i + 1, COL_STATUS + 1).setValue('投稿済');
      sheet.getRange(i + 1, COL_TWEETID + 1).setValue(tweetId);
      return; // 1回の実行で1件だけ
    }
  }
}
```

1回のトリガー実行で1件だけ処理するのがポイントです。GASの実行時間制限（6分）に引っかからないようにしています。
